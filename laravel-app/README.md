# Oil Holding Dashboard — Управление Холдингом & Аналитика Добычи 

Веб-приложение на базе фреймворка **Laravel**, разработанное для автоматизации учета персонала, визуализации внутренней иерархии нефтегазового холдинга и операционного мониторинга суточных производственных показателей фонда скважин.

---

##  Стек технологий

* **PHP 8.4** — актуальная версия серверного языка с высокой производительностью.
* **Laravel 11.x / 12.x** — современный масштабируемый MVC-фреймворк.
* **PostgreSQL** — промышленная реляционная СУБД для надежного хранения данных холдинга.
* **Blade + JavaScript** — динамический рендеринг интерфейсов дашборда на стороне сервера с интерактивными вкладками.
* **Faker / Factories** — автоматизированная генерация комплексных тестовых наборов данных.
* **Composer** — менеджер пакетов и автозагрузки PHP классов.

---

##  Архитектура холдинга и функционал

1. **Многоуровневая бизнес-логика (Иерархия):**
    * **Нефтяные компании (`companies`):** Материнские структуры холдинга (генерируются в количестве 22 предприятий).
    * **Сотрудники (`employees`):** Персонал, жестко закрепленный за соответствующими дочерними компаниями.
    * **Буровые скважины (`wells`):** Производственный фонд, привязанный к месторождениям конкретных компаний холдинга.
    * **Суточная добыча (`daily_productions`):** Исторические логи производственных показателей (дебит жидкости, обводненность) по каждой скважине за последние 365 дней.

2. **Интерактивный Дашборд:**
    * Главный экран управления на базе шаблона `dashboard.blade.php` с разделением информации через систему вкладок (Tabs).
    * Панель навигации (`sidebar.blade.php`) для переключения контекста холдинга.
    * Модульный вывод списков персонала холдинга через подшаблоны (`content-emps.blade.php` и `content-wells.blade.php`).

3. **Оптимизация базы данных:**
    * Связи «Один ко многим» (*HasMany / BelongsTo*) с каскадным удалением (*onDelete('cascade')*).
    * Кастомная консольная команда `php artisan make:data` для быстрой генерации всей структуры аналитики холдинга «в один клик».

##  Математическая модель расчета дебита нефти

В модуле аналитики скважин реализован автоматический расчет ключевых технологических показателей на основе суточных логов добычи.

1. **Дебит жидкости ($Q_ж$, $m^3/сут$):** Берётся напрямую из показаний приборов учета (записывается в базу как `production_value`).
2. **Дебит чистой нефти ($Q_н$, $т/сут$):** Рассчитывается на основе дебита жидкости, пластовой обводненности ($W$, %) и средней плотности нефти ($\rho_н \approx 0.85 \, т/м^3$) по формуле:

$$Q_н = Q_ж \times \left(1 - \frac{W}{100}\right) \times \rho_н$$

Данный расчет производится динамически на стороне сервера при формировании графиков и аналитических таблиц для вывода преподавателю.

##  Структура Базы Данных (Схема PostgreSQL)

Проект разворачивает 4 взаимосвязанные таблицы:

### 1. Таблица `companies` (Нефтяные компании)
* `id` (BigInt, Primary Key) — Уникальный идентификатор компании.
* `name` (String) — Название предприятия холдинга (например, "Жайык Ойл").
* `timestamps` (`created_at`, `updated_at`).

### 2. Таблица `employees` (Штат сотрудников)
* `id` (BigInt, Primary Key).
* `company_id` (Foreign Key) — Привязка к таблице `companies` с каскадным удалением.
* `first_name` (String) — Имя сотрудника.
* `last_name` (String) — Фамилия сотрудника.
* `timestamps`.

### 3. Таблица `wells` (Фонд буровых скважин)
* `id` (BigInt, Primary Key).
* `company_id` (Foreign Key) — Привязка к материнской компании.
* `name` (String) — Технологический номер скважины (например, "Скважина №402").
* `timestamps`.

### 4. Таблица `daily_productions` (Суточные логи добычи)
* `id` (BigInt, Primary Key).
* `well_id` (Foreign Key) — Привязка к конкретной скважине.
* `date` (Date) — Дата снятия замеров.
* `production_value` (Decimal, 10, 2) — Суточный дебит жидкости ($m^3$).
* `water_cut` (Decimal, 5, 2) — Процент обводненности продукции (от 0% до 100%).
* `timestamps`.
